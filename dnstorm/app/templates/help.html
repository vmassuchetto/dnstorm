  • {% trans "Writing a good problem description" %}

    {% captureas button %} {% endcaptureas %} {% captureas _button %}{% endcaptureas %} {% captureas left %}{% endcaptureas %}

    {% blocktrans %}{{ left }}When you {{ button }}post a problem{{ _button }} you become its manager, being capable of selecting the problem's permissions and invite users to contribute. A good problem description is objective, properly introduced, summarized, with the relevant points cleared and well specified. There's no magic formula, but we can give you some tips about writing problems:{% endblocktrans %}

    • {% trans "Research before writing. And research serious. You have to understand the problem." %}
    • {% trans "Try to use well the web editor while writing. People will understand better the text if it uses headings, references, citations, lists, paragraphs separated properly, links, images, and so." %}
    • {% trans "Proof-read before posting. Grammar is important." %}
    • {% trans "Be open for questioning when the problem is published. Reply the comments and analyze each of the contributor's considerations." %}
